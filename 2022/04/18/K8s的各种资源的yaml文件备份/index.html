考虑到k8s集群并没有对ingress、configmap等配置文件备份的功能，因此为目前k8s集群设置定时备份。采用github上的项目GitHub - pieterlange/kube-backup: Kubernetes resource state sync to git

其作用是在k8s集群里创建一个cronjob，定时执行每隔多少时间备份“ingress deployment configmap svc rc ds customresourcedefinition networkpolicy statefulset storageclass cronjob”等资源的yaml文件到gitlab中。

操作如下：

1.在gitlab中创建项目kube-backup以及创建账号k8sbackup并给账号添加ssh公钥以及对项目kube-backup的Maintainer权限，最后新建k8s命名空间kube-backup并在命名空间中添加ssh私钥

2.git clone项目，修改其中的yaml的配置，最后应用它们到命名空间kube-backup中：

最后可以看到gitlab上,已经同步的k8s的资源yaml的情况
